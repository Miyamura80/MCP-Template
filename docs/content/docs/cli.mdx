## Installation

After installing the package, the `mycli` command is available:

```bash
mycli --help
```

If you installed from source, prefix with `uv run`:

```bash
uv run mycli --help
```

## Global Flags

Flags go **before** the subcommand:

```bash
mycli [flags] <command> [args]
```

| Flag | Short | Description |
|---|---|---|
| `--verbose` | `-v` | Increase output verbosity |
| `--quiet` | `-q` | Suppress non-essential output |
| `--debug` | | Show full tracebacks on error |
| `--format` | `-f` | Output format: `table` (default), `json`, `plain` |
| `--dry-run` | | Preview actions without executing |
| `--version` | `-V` | Print version and exit |

## Commands

### greet

Greet someone by name.

```bash
mycli greet Alice
# Hello, Alice!

mycli greet Alice --shout
# HELLO, ALICE!

mycli greet Bob --shout --times 3
```

### config

Inspect and modify configuration.

```bash
mycli config show                           # view full config
mycli config get llm_config.cache_enabled   # get a single value
mycli config set logging.verbose false      # set an override
```

### doctor

Run health checks on the project environment.

```bash
mycli doctor          # check everything
mycli doctor --fix    # auto-fix what it can
```

### completions

Install shell tab-completions for faster usage.

```bash
mycli completions install bash
mycli completions install zsh
mycli completions install fish
```

### update

Check for a newer version and upgrade.

```bash
mycli update
```

### telemetry

Manage anonymous usage telemetry.

```bash
mycli telemetry status     # show enabled/disabled
mycli telemetry disable    # opt out
mycli telemetry enable     # opt back in
```

You can also set `CLI_TELEMETRY_DISABLED=1` to disable.

## Output Formats

The `--format` flag controls how data is rendered:

```bash
mycli config show                    # table (default)
mycli --format json config show      # machine-readable JSON
mycli --format plain config show     # simple key: value lines
```

JSON output is useful for piping into other tools:

```bash
mycli --format json config show | jq '.llm_config'
```

## Error Handling

By default, errors show a concise message. Use `--debug` for full tracebacks:

```bash
mycli --debug config get nonexistent.key
```

## Dry Run

Preview what a command would do without executing it:

```bash
mycli --dry-run greet Alice
# [DRY RUN] Would greet Alice
```
