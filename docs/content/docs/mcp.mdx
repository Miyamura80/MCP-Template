The MCP server lets AI assistants call your tools directly. Set it up once and Claude (or any MCP-compatible client) can use all available tools without manual copy-pasting.

## Setup

Replace `/path/to/MCP-Template` with the actual path to your cloned repo.

<Tabs items={['Claude Desktop', 'Claude Code', 'Cursor', 'VS Code']} groupId="mcp-client">
<Tab value="Claude Desktop">

Add to your Claude Desktop config file:

- **macOS**: `~/Library/Application Support/Claude/claude_desktop_config.json`
- **Windows**: `%APPDATA%\Claude\claude_desktop_config.json`
- **Linux**: `~/.config/Claude/claude_desktop_config.json`

```json
{
  "mcpServers": {
    "mycli": {
      "command": "uv",
      "args": ["--directory", "/path/to/MCP-Template", "run", "mycli-mcp"],
      "env": {}
    }
  }
}
```

Restart Claude Desktop after saving the config.

### Other MCP Clients

The server uses **stdio transport** (stdin/stdout). Any MCP-compatible client can connect by spawning `uv --directory /path/to/MCP-Template run mycli-mcp` as a subprocess.

## Available Tools

Once connected, the AI assistant has access to these tools:

| Tool | Description | Parameters |
|------|-------------|------------|
| `greet` | Greet a user by name | `name` (str), `shout` (bool, optional), `times` (int, optional) |
| `config_show` | Show the full configuration | _(none)_ |
| `config_get` | Get a single config value | `key` (str) |
| `config_set` | Set a config override | `key` (str), `value` (str) |
| `doctor` | Run project health checks | `fix` (bool, optional) |

## Example Conversations

After setup, you can ask Claude things like:

- "Greet Alice with a shout" - Claude calls the `greet` tool with `{"name": "Alice", "shout": true}`
- "What's the current LLM config?" - Claude calls `config_get` with `{"key": "llm_config"}`
- "Run a health check and fix any issues" - Claude calls `doctor` with `{"fix": true}`

### What Claude Sees

**Input:**
```json
{ "name": "Alice", "shout": true, "times": 1 }
```

**Result:**
```json
{ "message": "HELLO, ALICE!", "times": 1 }
```

## Error Handling

If a tool call fails, the AI client receives an error message and can adjust:

```json
{ "error": "Key not found: nonexistent.key" }
```

## Debugging

Use the [MCP Inspector](https://github.com/modelcontextprotocol/inspector) to test tools interactively without an AI client:

```bash
npx @modelcontextprotocol/inspector uv --directory /path/to/MCP-Template run mycli-mcp
```

This opens a web UI where you can call each tool, inspect parameters, and see responses.
