## Overview

Services are pure business logic functions that are automatically exposed through all three interfaces: **CLI**, **REST API**, and **MCP**. Write your logic once as a service and it becomes available everywhere.

```
┌─────────┐   ┌───────────┐   ┌────────────┐
│   CLI   │   │  REST API │   │ MCP Server │
│ (Typer) │   │ (FastAPI) │   │ (FastMCP)  │
└────┬────┘   └─────┬─────┘   └─────┬──────┘
     │              │               │
     └──────────────┼───────────────┘
                    │
            ┌───────┴────────┐
            │ Service Layer  │
            │ (Pure Python)  │
            └───────┬────────┘
                    │
            ┌───────┴────────┐
            │ Pydantic I/O   │
            │    Models      │
            └────────────────┘
```

Each service has:
- A **name** used as the CLI command, API route, and MCP tool name
- **Input/output models** (Pydantic) for validation and serialization
- A **pure function** containing the business logic

## Defining a Service

### 1. Create the Models

Define Pydantic input and output models in `models/`:

```python
# models/greet.py
from pydantic import BaseModel

class GreetInput(BaseModel):
    name: str
    shout: bool = False
    times: int = 1

class GreetResult(BaseModel):
    message: str
    times: int = 1
```

### 2. Implement the Service

Create a service in `services/` using the `@service` decorator:

```python
# services/greet.py
from models.greet import GreetInput, GreetResult
from services import service

@service(
    name="greet",
    description="Greet a user by name",
    input_model=GreetInput,
    output_model=GreetResult,
)
def greet(input: GreetInput) -> GreetResult:
    message = f"Hello, {input.name}!"
    if input.shout:
        message = message.upper()
    return GreetResult(message=message, times=input.times)
```

### 3. Use from a CLI Command

CLI commands import and call the service directly:

```python
# commands/greet.py
from models.greet import GreetInput
from services.greet import greet

def main(name: str, shout: bool = False, times: int = 1) -> None:
    """Greet someone by name."""
    result = greet(GreetInput(name=name, shout=shout, times=times))
    typer.echo(result.message)
```

The MCP and API interfaces register services automatically - no extra wiring needed.

## How Each Interface Exposes Services

| Interface | Discovery | Route/Tool | Auth |
|-----------|-----------|------------|------|
| **CLI** | Commands import services directly | `mycli greet Alice` | None |
| **REST API** | Auto-registered from registry | `POST /api/v1/services/greet` | API key or OAuth |
| **MCP** | Auto-registered from registry | Tool: `greet` | None (Claude trust model) |

## Service Registry

The `@service` decorator registers functions in a global registry (`services/__init__.py`). Both the API server and MCP server iterate over this registry at startup to create routes/tools dynamically.

```python
from services import get_registry

for entry in get_registry():
    print(f"{entry.name}: {entry.description}")
    # entry.input_model  → Pydantic class
    # entry.output_model → Pydantic class
    # entry.func         → callable
```

## Built-in Services

| Service | Description | Input | Output |
|---------|-------------|-------|--------|
| `greet` | Greet a user by name | `name`, `shout`, `times` | `message`, `times` |
| `config_show` | Show full project configuration | _(none)_ | `config` (dict) |
| `config_get` | Get a single config value | `key` | `key`, `value` |
| `config_set` | Set a configuration override | `key`, `value` | `key`, `coerced_value` |
| `doctor` | Run project health checks | `fix` (bool) | `checks`, `has_failures` |

## Adding a New Service

import { Step, Steps } from 'fumadocs-ui/components/steps';

<Steps>
<Step>

### Define Models

Create `models/my_feature.py` with Pydantic `Input` and `Result` classes.

</Step>
<Step>

### Implement Logic

Create `services/my_feature_svc.py` with the `@service` decorator. Keep it pure - no HTTP, no CLI, no transport awareness.

</Step>
<Step>

### Wire CLI (optional)

Create `commands/my_feature.py` that imports and calls your service. The CLI command is auto-discovered.

</Step>
<Step>

### Register for MCP/API

Add an import to `mcp_server/server.py` in `_register_tools()`:

```python
import services.my_feature_svc  # noqa: F401
```

The API server discovers services automatically via `pkgutil`, so no changes are needed there.

</Step>
</Steps>
